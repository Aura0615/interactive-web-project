Method Overview

执行过程概览：从研究判断到回流校验

我们不是把 AI 当作一次性问答工具，而是把行业研究拆解为一个由人类主导、AI 分层执行、Agent/Skill 承接任务、结果回流校验的循环式研究系统。

01

人类定向

团队先确定八个研究维度、分析边界和最终路演目标，保证 AI 执行不偏题。

02

AI 拆解

利用 GPT 将每个维度拆解为 3–4 个可执行研究环节，把宏观任务转化为清晰动作。

03

Skill 封装

通过 GPT + Codex 将研究环节转化为 .yaml Skill、执行框架和 Python 文件。

04

Agent 执行

由 OpenClaw 与阿里云百炼 / JVS Claw 调用 Skill，完成资料处理、结构化分析与部分可视化输出。

05

回流校验

结果经 Agent 打包、GPT 总结和人工审查后回到团队，形成可迭代的研究闭环。

Method Map 04

循环式 AI Agent 研究工作流

传统工作流往往是线性推进，而我们的流程强调“任务拆解—Skill 执行—结果回流—人工校验”的循环结构。AI 负责提高执行效率，人负责稳住研究方向和最终判断。

01

研究方向设定

先定方向和判断标准，再让 AI 进入执行层。

02

维度任务拆解

宏观问题被转化为可执行、可复查的研究环节。

03

Skill 规则封装

将重复动作沉淀下来，让任务规则固定。

04

Agent 调用执行

Agent 提高执行效率，但不直接替代研究判断。

05

结果回流整理

输出不直接进入结论，而是先回到分析链路中。

06

人工审查与迭代

最终判断留在人手里，工作流继续迭代。

研究方向设定

根据赛题、行业资料和报告目标，把研究对象与路演表达先收束到清晰边界。

AI 如何参与：根据赛题、行业资料和报告目标，辅助生成研究维度建议。
产出物：八个研究维度与每个维度的分析边界。
人工把关：团队决定哪些维度保留、删减或合并。
Method Map 03

工作流闭环：人管 AI，AI 管下一层 AI

这部分单独表现方法论里的循环逻辑：人负责顶端方向与最终取舍，AI 负责拆解和管理下一环节 AI，底层 Agent 输出再反向回流，经过整理、总结后回到人的顶端判断，继续修正下一轮任务。

Execution System

基于 Skill 封装的 Agent 执行系统

GPT 已经在前序环节完成八个研究维度和具体执行环节的拆解；随后我们使用 GPT + Codex 将这些执行环节封装成 Skill 固定任务规则，由 OpenClaw 与阿里云百炼 / JVS Claw 调用，辅助完成资料整理、研究分析、可视化转译和质量检查，结果再进入 GPT 总结与人工校验。

Fragrance Research Agent

香氛个护行业研究 Agent

该 Agent 接收已经拆解好的研究维度与执行环节，按照“资料整理—研究分析—可视化转译—质量检查—人工校验”的路径运行，辅助团队完成结构化资料、章节初稿和路演素材生成。

01 接收已拆解的研究维度与任务说明
02 调用资料整理 Skill
03 调用研究分析 Skill
04 调用可视化转译 Skill
05 调用质量检查 Skill
06 输出结果并进入 GPT 总结与人工校验
Skill 01

Source Structuring

资料整理 Skill接收行业资料、公开报告、人工搜索材料和已有数据，将分散资料按八个研究维度归类，提取有效信息、来源线索和可用章节。

Skill 02

Research Execution

研究分析 Skill基于已整理资料，围绕行业定义、行业分类、PEST、用户画像、价格体系、差异化、痛点需求和渠道矩阵等维度，生成结构化分析和章节初稿。

Skill 03

Visualization Translation

可视化转译 Skill将研究过程和分析结论转化为热力图、流程图、力导向图、闭环图和路演网页中的可视化表达，服务 5 分钟现场讲解。

Skill 04

Quality Check

质量检查 Skill检查 AI 输出是否存在空话、逻辑跳跃、来源不明、工具堆砌或缺少人工判断，并给出修改建议，确保最终结果能够进入 GPT 总结和人工审查环节。

Method Value

方法价值：从线性执行到循环研究系统

这套工作流的价值不在于让 AI 替代人工，而是把复杂行研任务拆成清晰环节，并通过 Skill 封装、Agent 执行和人工回流校验，形成一个可拆解、可复用、可校验、可迭代的研究系统。

人主导，AI 分层执行 输出回流校验，研究持续迭代
人类定向 确定研究方向、维度边界和最终判断标准。
AI 拆解 GPT 将八个行研维度拆解为 3–4 个可执行环节。
Skill 封装 GPT + Codex 将执行环节转化为 .yaml Skill、框架和 Python 代码。
Agent 执行 OpenClaw 与阿里云百炼 / JVS Claw 调用 Skill 执行资料整理、结构化分析和部分可视化输出。
结果整理 Agent 将原始输出打包并分析，再交给 GPT 总结。
人工审查 人对结果进行判断、筛选和修正，并决定是否进入下一轮迭代。
Selected Value

可迭代

传统 AI 使用往往停留在“一次提问—一次输出”的线性模式，而我们的工作流将结果重新回流到 GPT 总结和人工审查环节。人工根据输出质量调整研究方向、Prompt 或 Skill，使整个流程能够不断修正，而不是一次性结束。
