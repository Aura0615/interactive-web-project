Method Overview

执行过程概览：从研究判断到回流校验

我们不是把 AI 当作一次性问答工具，而是把行业研究拆解为一个由人类主导、AI 分层执行、Agent/Skill 承接任务、结果回流校验的循环式研究系统。

01

人类定向

团队先确定八个研究维度、分析边界和最终路演目标，保证 AI 执行不偏题。

02

AI 拆解

利用 GPT 将每个维度拆解为 3–4 个可执行研究环节，把宏观任务转化为清晰动作。

03

Skill 封装

通过 GPT + Codex 将研究环节转化为 .yaml Skill、执行框架和 Python 文件。

04

Agent 执行

由 OpenClaw 与阿里云百炼 / JVS Claw 调用 Skill，完成资料处理、结构化分析与部分可视化输出。

05

回流校验

结果经 Agent 打包、GPT 总结和人工审查后回到团队，形成可迭代的研究闭环。

Method Map 02

循环式 AI Agent 研究工作流

传统工作流往往是线性推进，而我们的流程强调“任务拆解—Skill 执行—结果回流—人工校验”的循环结构。AI 负责提高执行效率，人负责稳住研究方向和最终判断。

01

研究方向设定

先定方向和判断标准，再让 AI 进入执行层。

02

维度任务拆解

宏观问题被转化为可执行、可复查的研究环节。

03

Skill 规则封装

将重复动作沉淀下来，让任务规则固定。

04

Agent 调用执行

Agent 提高执行效率，但不直接替代研究判断。

05

结果回流整理

输出不直接进入结论，而是先回到分析链路中。

06

人工审查与迭代

最终判断留在人手里，工作流继续迭代。

研究方向设定

根据赛题、行业资料和报告目标，把研究对象与路演表达先收束到清晰边界。

AI 如何参与：根据赛题、行业资料和报告目标，辅助生成研究维度建议。
产出物：八个研究维度与每个维度的分析边界。
人工把关：团队决定哪些维度保留、删减或合并。
Method Map 03

工具—任务—成果力导向图：从线性流程到协作网络

这张图展示的不是工具堆叠，而是一个多层级协作网络：人负责 harness，GPT 负责拆解与封装，Codex 负责代码化，OpenClaw 与阿里云百炼负责执行，结果再回流给 GPT 和人类审查。

协作网络，而非工具清单

我们将 AI 工具分成三类：上层模型负责拆解与封装，执行型 Agent 负责调用 Skill 处理任务，人工负责方向判断和最终校验。这样做的意义不是追求一键完成，而是让每一步输出都能被追踪、解释和修正。

Execution System

基于 Skill 封装的 Agent 执行系统

我们将每个研究环节封装为可调用 Skill，使 Agent 不再依赖一次性对话，而是按照明确的输入、处理逻辑、输出格式和校验规则执行任务。

Fragrance Research Agent

香氛个护行业研究 Agent

该 Agent 接收行业研究任务与已有资料，按照“维度拆解—Skill 调用—结果整理—人工校验”的路径运行，辅助团队完成资料结构化、章节初稿和可视化素材生成。

01 接收研究维度与任务说明
02 调用资料整理 Skill
03 调用维度分析 Skill
04 调用可视化转译 Skill
05 输出结果并进入人工校验
Skill 01

Dimension Decomposition

将八个行业研究维度拆解为具体执行环节，明确每一环节的输入资料、分析动作和输出形式。

Skill 02

Research Execution

面向行业定义、分类、PEST、用户、价格、差异化、痛点与渠道等维度，执行资料整理和结论生成。

Skill 03

Visualization Translation

将研究过程与分析结果转化为热力图、流程图、力导向图和路演网页中的可视化表达。

Skill 04

Quality Check

检查结论是否存在空话、逻辑跳跃、工具滥用或缺少人工判断，并给出修改建议。

Proof Chain

过程证据：从 Prompt 到 Skill 的可追溯链路

这部分不展示过多细节，而是证明过程真实发生过，并说明我们知道如何校验输出。

Prompt 迭代证据

展示从普通提问到结构化任务指令的变化，证明 AI 使用经过设计，而不是简单问答。

Skill 配置证据

展示 .yaml Skill、执行框架或代码片段，证明研究环节被封装为可复用能力。

输出校验证据

展示一次“原始输出 → GPT 总结 → 人工修改”的回流过程，证明最终结论经过人工审查。

Method Value

方法价值：让 AI 研究过程可拆解、可复用、可校验

这套工作流的价值并不在于取代人工，而在于将原本分散的资料搜索、框架生成、内容整理和可视化转译，变成一个可以被拆解、复用和检查的研究系统。人类负责方向和判断，AI 负责降低重复劳动成本，Agent 负责承接结构化执行。

三层价值

方法不是为了显得复杂，而是为了让复杂行研任务可以被交接、被复盘、被修正。每一次输出都能回到“输入是什么、规则是什么、谁做判断”的链路中。

可拆解：把宏观行研任务拆成清晰环节 可复用：把重复动作沉淀为 Skill 可校验：让输出经过回流与人工审查

现场结论

我们的展示重点不是“用了多少工具”，而是如何把人类判断、模型拆解、Agent 执行和人工校验组织成一个轻量级研究型 Agent 工作流。
